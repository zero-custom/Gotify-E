# i18n — Multi-Language UI Layer

The gateway injects a client-side i18n layer into proxied Gotify HTML pages. It translates the Gotify frontend UI and its relative timestamps into the browser's preferred language without modifying the backend.

## Architecture

```
proxy.py:inject_i18n()          Server-side: injects <script> tags
    │
    ├─ /_gateway/i18n.js         Engine — language detection, text walking,
    │                              relative time translation (stateless)
    ├─ /_gateway/enhance.js       Enhance — brand, layout, time toggle,
    │                              bootstraps i18n (stateful)
    └─ /_gateway/lang/{lang}.js  Static locale data — `__LOCALE_MAP` array
                                    (pure data, no logic)
```

**i18n.js** provides the passive engine (detect, walk, replace). **enhance.js** consumes it along with its own features (brand marker, time toggle, layout CSS). The translation files (`zh_CN.js`, `fr.js`, etc.) are pure arrays of `[en, translated]` tuples.

## Language Detection (priority order)

1. **`?lang=` query parameter** — highest priority, e.g. `?lang=fr`
2. **`localStorage` key `gotify-lang`** — persists across sessions
3. **`navigator.language` / `navigator.userLanguage`** — browser preference

### Automatic browser detection

The engine normalises locale codes to translation file names:

| Browser locale | Translation file |
|---|---|
| `zh`, `zh-CN`, `zh-SG`, `zh-Hans` | `zh_CN.js` |
| `fr`, `fr-FR`, `fr-CA`, `fr-CH`, `fr-BE` | `fr.js` |
| `de`, `de-DE`, `de-AT`, `de-CH` | `de.js` |
| `es`, `es-ES`, `es-MX`, `es-AR` | `es.js` |
| `pt`, `pt-PT`, `pt-BR` | `pt.js` |
| `ru`, `ru-RU` | `ru.js` |
| `it`, `it-IT`, `it-CH` | `it.js` |
| `ko`, `ko-KR` | `ko.js` |
| `ja`, `ja-JP` | `ja.js` |
| Everything else | Falls back to English (no translation) |

## Translation Mechanism

Two independent passes:

### Pass 1 — Static UI text (`__LOCALE_MAP`)

Each locale file exposes `window.__LOCALE_MAP`, an array of `[english, translated]` tuples:

```js
window.__LOCALE_MAP = [
  ['Delete All', '删除全部'],
  ['Push Message', '推送消息'],
  // … 180 entries per language
];
```

`walkReplace()` walks the DOM text nodes and performs whole-word replacements using the locale map. Entries are sorted longest-first to prevent partial matches. Only text outside `SCRIPT`, `STYLE`, `INPUT`, `TEXTAREA`, and `.notranslate` elements is touched.

### Pass 2 — Relative time (`Intl.RelativeTimeFormat`)

`i18n.js` parses Gotify's English relative time strings (produced by `react-timeago` with `locale:'en'`) and converts them via `Intl.RelativeTimeFormat`.

The `ENG_UNIT` dictionary maps English unit words/abbreviations to Intl unit identifiers. No hand-written translations are needed for relative times — the browser's built-in `Intl` API handles all 9 languages.

## Adding a New Language

1. Create `app/_gateway/lang/{code}.js` with the same `window.__LOCALE_MAP` format (180 entries from the existing files)
2. Add normalisation entry in `i18n.js:NORM` object
3. Add browser detection branch in `i18n.js:detectLang()` `navigator.language` block

## Static File Serving

The gateway mounts `app/_gateway/` as a static directory at `/_gateway/`:

```python
app.mount("/_gateway", StaticFiles(directory=str(_GATEWAY_DIR)), name="gateway")
```

Translation files under `app/_gateway/lang/` are fetched dynamically by `i18n.js:activate()`:
- On success: `__LOCALE_MAP` is populated, `Intl.RelativeTimeFormat` is created, `walkReplace()` runs
- On 404: silently falls back to English, clears the `gotify-lang` localStorage key
